현재 브랜치 작업에 맞춰 Claude 문서와 프로젝트 문서를 업데이트합니다.

1. `git status`와 변경 파일을 확인합니다.
2. 변경이 시나리오, embed, simulator, setup/config 중 어디에 속하는지 분류합니다.
3. 필요한 경우 루트 `CLAUDE.md`, README, CONTRIBUTING을 갱신합니다.
4. 신규 주요 feature slice가 생겼다면 해당 디렉토리에 `docs/CLAUDE.md`를
   추가할지 판단합니다.
5. 문서 변경이 필요 없으면 한국어로 이유를 설명합니다.

문서는 실제 코드와 어긋나지 않게 짧고 유지보수 가능하게 작성합니다.
